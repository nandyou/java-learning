Spring MVC Controller

A Spring Controller is a class that handles incoming HTTP requests and returns responses. It sits at the entry point of your web layer, routing requests to the right logic.

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.ui.Model;
 
@Controller
@RequestMapping ("/products")
public class ProductController {
 
// ① Dependency injection via constructor (preferred)
private final ProductService productService;
 
@Autowired
public ProductController (ProductService productService) {
this.productService = productService;
}
 
// ② Return a view name (Thymeleaf / JSP template)
@RequestMapping("/list")
public String listProducts( Model model) {
model.addAttribute("products" , productService.findAll());
return "products/list" ;
}
 
// ③ Write directly to HTTP response body
@ResponseBody
@RequestMapping("/count")
public String count() {
return "Total: " + productService.count();
}
 
// ④ Handle exceptions in this controller
@ExceptionHandler (ProductNotFoundException.class)
@ResponseBody
public String handleNotFound( ProductNotFoundException ex) {
return "Error: " + ex.getMessage();
}
}

@RestController — the modern shortcut that combines @Controller + @ResponseBody on every method. Use this for REST APIs.

@RestController
@RequestMapping ("/api/products")
public class ProductApiController {
// Every method automatically writes JSON to the response body
// No need for @ResponseBody on each method
}
@Controller
Marks this class as a Spring MVC controller — a stereotype annotation.
Spring automatically detects it during component scanning and registers it as a bean.
Use @RestController for REST APIs — it adds @ResponseBody to every method.
@RequestMapping at class level
Sets a base URL prefix for all methods in this controller.
Here, every method URL starts with /products.
Method-level mappings are appended: /products + /list = /products/list
Controller class naming
Convention: name ends with Controller, matches the resource it manages.
One controller per resource type keeps code organised and focused.
Place in a controller or web package alongside service and repository layers.
@Autowired (constructor injection)
Tells Spring to inject a ProductService bean when creating this controller.
Constructor injection is preferred over field injection — it makes dependencies explicit and testable.
If there's only one constructor, @Autowired is optional in Spring 4.3+.
Returning a String (View Name)
When a method returns a String (without @ResponseBody), Spring treats it as a view name.
A ViewResolver (e.g. Thymeleaf) resolves it to an actual template file.
Return "redirect:/path" or "forward:/path" for redirects.
Model
A map-like container that carries data from the controller to the view template.
model.addAttribute("key", value) makes data available in the template as ${key}.
Spring automatically injects Model when declared as a method parameter.
View Name String
This resolves to a template file at src/main/resources/templates/products/list.html (Thymeleaf).
The prefix/suffix are configured in application.properties.
Default Thymeleaf: prefix = classpath:/templates/, suffix = .html
@ResponseBody
Tells Spring to write the return value directly into the HTTP response body — no view resolution.
Combined with Jackson on the classpath, objects are automatically serialised to JSON.
Eliminates the need for @ResponseBody on every method when using @RestController.
@ExceptionHandler
Catches a specific exception thrown by any method in this controller.
Returns a custom error response instead of a 500 page.
For global exception handling across all controllers, use @ControllerAdvice.
@RestController
A convenience annotation = @Controller + @ResponseBody on the class.
Every method's return value is written to the HTTP response as JSON (or XML).
The standard choice for building REST APIs in Spring Boot.
