String Operations

Strings in Java are immutable objects with a rich API. Every operation returns a new String — the original is never changed.

public class StringDemo {
public static void main(String[] args) {
 
String s = "Hello, Java World!";
 
// ① Length and character access
int len = s. length (); // 18
char ch = s. charAt (7); // 'J'
 
// ② Searching
int idx = s. indexOf ("Java"); // 7
boolean has = s. contains ("World"); // true
boolean sw = s. startsWith ("Hello"); // true
boolean ew = s.endsWith("!"); // true
 
// ③ Extracting parts
String sub = s. substring (7, 11); // "Java"
String[] parts = s. split (", "); // ["Hello", "Java World!"]
 
// ④ Transforming
String up = s. toUpperCase ();
String low = s.toLowerCase();
String tr = " hello " . strip (); // "hello"
String rep = s. replace ("Java", "Python");
 
// ⑤ Comparison — never use == for Strings!
String a = "hello";
String b = new String("hello");
boolean eq1 = a. equals (b); // true ✓
boolean eq2 = a == b; // false ✗
boolean ic = a. equalsIgnoreCase ("HELLO"); // true
 
// ⑥ Building strings efficiently
StringBuilder sb = new StringBuilder();
sb.append("Hello").append(" ").append("World");
String result = sb.toString();
}
}
.length()
Returns the number of characters in the string. It's a method call (with parentheses).
Don't confuse with arrays where .length is a field (no parentheses).
An empty string "" has length 0, not null.
.charAt(index)
Returns the character at the given index. Zero-based.
Returns a primitive char, not a String.
Throws StringIndexOutOfBoundsException for invalid index.
.indexOf(str)
Returns the starting index of the first occurrence of the given substring.
Returns -1 if not found — always check for -1 before using the result!
Use .lastIndexOf() to find the last occurrence.
.contains(str)
Returns true if the string contains the given sequence anywhere.
Cleaner than indexOf() != -1 when you just need a yes/no answer.
Case-sensitive — "world" won't match "World".
.startsWith() / .endsWith()
Check if the string begins or ends with a given prefix or suffix.
Very useful for file extension checks: filename.endsWith(".pdf").
Both are case-sensitive. Use .toLowerCase() first for case-insensitive check.
.substring(start, end)
Extracts characters from index start (inclusive) to end (exclusive).
"Hello".substring(1, 4) → "ell"
Omit end to go to the end of the string: s.substring(7)
.split(regex)
Splits the string by the given delimiter, returning a String[].
The delimiter is a regular expression — escape special chars: split("\\.") for dots.
Use String.join(", ", array) to reverse the split.
.toUpperCase() / .toLowerCase()
Returns a new string with all characters converted. The original is unchanged.
Best practice: always use the locale-aware version for user-facing text.
toUpperCase(Locale.ROOT) for consistent behaviour across regions.
.strip() vs .trim()
Both remove leading and trailing whitespace. strip() is preferred (Java 11+).
strip() correctly handles Unicode whitespace; trim() only handles ASCII ≤ 32.
Also see: stripLeading() and stripTrailing().
.replace(old, new)
Replaces all occurrences of old with new. Returns a new String.
For pattern-based replacement use .replaceAll(regex, replacement).
Since strings are immutable, you must capture the returned value.
.equals(other)
Compares content character by character. This is the correct way to compare strings.
Defined in the String class to compare values, not memory addresses.
If other could be null, use Objects.equals(a, b) to avoid NullPointerException.
== on Strings — DON'T!
== compares object references (memory addresses), not content.
Two separate String objects with the same text will return false with ==.
It may accidentally return true for string literals (due to interning) — never rely on this!
.equalsIgnoreCase()
Compares content while ignoring upper/lowercase differences.
"Java".equalsIgnoreCase("JAVA") → true
Ideal for user input comparison where casing shouldn't matter.
StringBuilder
A mutable string buffer for efficient building of long strings in loops.
Each + on a regular String creates a new object — slow in loops!
StringBuilder modifies in-place, then call .toString() at the end.
Use StringBuffer instead if you need thread-safety.
