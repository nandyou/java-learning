Java Class Structure

Every Java program is built from classes. Here's the anatomy of a complete class.

package com.example.app;
 
import java.util.List;
 
public class BankAccount extends FinancialProduct {
 
// ① Fields (instance variables)
private String owner;
private double balance;
private static int totalAccounts = 0 ;
 
// ② Constructor
public BankAccount (String owner, double initialBalance) {
this .owner = owner;
this .balance = initialBalance;
totalAccounts++;
}
 
// ③ Methods
public void deposit( double amount) {
balance += amount;
}
 
// ④ Getter (accessor)
public double getBalance() {
return balance;
}
}
package
Groups related classes into a namespace, like folders.
Prevents naming conflicts — two classes can share a name if in different packages.
Convention: reverse domain name (com.company.module)
import
Brings another class into scope so you can use its short name.
Without import, you'd need to write the full path: java.util.List list;
import java.util.* imports all classes from a package (wildcard)
class keyword
Defines a new class — a blueprint for creating objects.
A class encapsulates state (fields) and behavior (methods).
One public class per .java file, matching the filename.
extends — Inheritance
BankAccount inherits all non-private fields and methods from FinancialProduct.
Java supports single inheritance only (one parent class).
Use implements for interfaces (multiple allowed).
Instance Fields
Variables declared inside the class but outside any method.
Each object (instance) gets its own copy of these fields.
Best practice: make fields private and provide getters/setters.
static modifier
Belongs to the class, not to individual objects.
All instances share one copy. Access via BankAccount.totalAccounts.
Static fields are initialized when the class is first loaded.
Constructor
A special method called when creating an object with new.
Same name as the class, no return type.
If you define none, Java provides a default no-arg constructor.
this keyword
Refers to the current object instance.
Used here to distinguish the field this.owner from the parameter owner.
Also used to call another constructor: this(arg)
Getter Method
Provides controlled read access to a private field.
Part of the encapsulation pattern in OOP.
Convention: get + FieldName (e.g. getBalance)
