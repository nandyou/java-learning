Access Modifiers

Access modifiers control who can see and use a class, field, or method.

public class AccessDemo {
 
// Accessible from ANYWHERE
public String publicName = "visible everywhere" ;
 
// Accessible only within THIS class
private String secretCode = "hidden" ;
 
// Accessible within package + subclasses
protected int sharedData = 42 ;
 
// No modifier = package-private (default)
double packageValue = 3.14 ;
 
// Methods with different access levels
public void publicMethod() {
// Can call private methods from within the same class
privateHelper();
}
 
private void privateHelper() {
// Internal implementation — hidden from outside
System.out.println(secretCode);
}
}

public class
The class itself is visible to all code in any package.
A public class file name must match the class name exactly.
Only one public class allowed per .java file.
public
The most permissive access level. Visible everywhere.
Use for APIs and methods you intend others to call.
Over-using public breaks encapsulation.
private
The most restrictive level. Only accessible within the same class.
This is the gold standard for fields — expose data via getters/setters.
Subclasses cannot access private members directly.
protected
Visible within the same package and to subclasses (even in other packages).
Used when building class hierarchies where child classes need access.
Common in template method patterns.
default (package-private)
No keyword — just omit the modifier.
Visible only to classes in the same package. Subclasses in other packages cannot see it.
Useful for internal implementation classes.
private method
Used for internal helper logic that shouldn't be part of the public API.
Allows you to refactor internals freely without breaking callers.
Private methods cannot be overridden in subclasses.
