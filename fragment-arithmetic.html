Arithmetic Operations

Java supports standard math operators, shorthand assignments, increment/decrement, and order-of-precedence rules.

public class ArithmeticDemo {
public static void main(String[] args) {
 
// ① Basic operators
int a = 10, b = 3;
 
int sum = a + b; // 13
int diff = a - b; // 7
int prod = a * b; // 30
int quot = a / b; // 3 (integer division — truncates!)
int rem = a % b; // 1 (remainder)
 
// ② Floating-point division
double exact = (double) a / b; // 3.3333...
 
// ③ Compound assignment operators
a += 5; // a = a + 5 → 15
a -= 2; // a = a - 2 → 13
a *= 3; // a = a * 3 → 39
a /= 4; // a = a / 4 → 9
a %= 4; // a = a % 4 → 1
 
// ④ Increment / Decrement
int x = 5;
x ++ ; // post-increment: use then add 1 → x = 6
-- x; // pre-decrement: subtract 1 then use → x = 5
 
// ⑤ Operator precedence
int result = 2 + 3 * 4; // 14, not 20
int forced = (2 + 3) * 4; // 20 — brackets first
 
// ⑥ Math utility
double root = Math .sqrt(144); // 12.0
double power = Math.pow(2, 10); // 1024.0
int abs = Math.abs(-42); // 42
}
}
+ Addition
Adds two numbers. Also used for String concatenation: "Hello" + " World".
When mixing types, the result is widened: int + double → double.
Avoid heavy + in loops — use StringBuilder instead.
- Subtraction
Subtracts the right operand from the left.
Also used as unary minus: -a negates the value.
Be careful with unsigned values — Java's int is signed.
* Multiplication
Multiplies two numbers. Result type follows the wider of the two operands.
Watch for overflow: int * int can exceed Integer.MAX_VALUE silently.
/ Division
When both operands are integers, result is truncated toward zero (no rounding).
10 / 3 = 3, not 3.33
Division by zero throws ArithmeticException!
Cast one operand to double for decimal result.
% Modulus (Remainder)
Returns what's left over after integer division.
Common uses: check even/odd (n % 2 == 0), wrap around (index % size).
Result keeps the sign of the left operand in Java.
Casting for float division
(double) a temporarily converts the int to a double before dividing.
This forces the division to produce a decimal result.
Alternatively write: a / (double) b or a / 3.0
Compound Assignment ( += )
Shorthand that combines an operation with assignment. All operators have compound forms:
+= -= *= /= %=
They also include an implicit narrowing cast: byte b; b += 1; compiles fine.
Post-increment ( x++ )
Returns the current value of x, then increments by 1.
int y = x++; → y gets old value, x increases.
Compare with pre-increment ++x: increments first, then returns.
Pre-decrement ( --x )
Decrements x by 1 first, then returns the new value.
int y = --x; → both y and x get the decreased value.
As a standalone statement x++ and ++x behave identically.
Operator Precedence
Java follows BODMAS/PEMDAS: * and / before + and -.
Use parentheses () to make intent explicit and override default order.
When in doubt, add brackets — clarity beats brevity.
Math class
java.lang.Math (auto-imported) provides common mathematical functions.
All methods are static — call them on the class directly.
Common: sqrt, pow, abs, max, min, floor, ceil, round, random
