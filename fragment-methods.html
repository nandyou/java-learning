Java Methods

Methods define the behavior of a class. Every method has a signature, body, and optional return value.

public class Calculator {
 
// ① Basic method anatomy
public int add ( int a , int b ) {
return a + b;
}
 
// ② void method — returns nothing
public void printResult( int result) {
System.out.println( "Result: " + result);
}
 
// ③ Method overloading
public double add ( double a , double b ) {
return a + b;
}
 
// ④ Static method — no instance needed
public static int square( int n) {
return n * n;
}
 
// ⑤ Varargs — variable argument count
public int sum( int... numbers) {
int total = 0 ;
for ( int n : numbers) total += n;
return total;
}
}
Access Modifier
Controls who can call this method.
public = anyone | private = this class only | protected = package + subclasses
Methods should generally be as restrictive as possible.
Return Type
Declares what type of value the method sends back to the caller.
The return statement must provide a value of this exact type.
Use void if the method returns nothing.
Method Name
Identifies the method. Should be a verb describing what it does.
Java convention: camelCase starting with a lowercase letter.
Examples: calculateTotal(), getUserById()
Parameters
Values passed into the method when it is called.
Format: type name. Multiple params separated by commas.
Java passes primitives by value, objects by reference.
void
Means the method does not return a value.
Used for methods that perform an action (like printing) rather than computing a result.
Cannot use the result of a void method in an expression.
Method Overloading
Same method name, different parameter types or count.
Java picks the right version at compile time based on the arguments you pass.
Return type alone is NOT enough to overload a method.
static method
Belongs to the class, not an instance. Call without creating an object:
Calculator.square(5)
Cannot access instance fields or call non-static methods directly.
Varargs ( int... )
Allows passing any number of arguments of the same type.
Inside the method, treated as an array.
Must be the last parameter. Only one vararg per method.
