POJO Classes

A Plain Old Java Object is a simple class with private fields, public getters/setters, a constructor, and utility methods — no frameworks or special rules.

public class Person {
 
// ① Private fields — encapsulated state
private String name;
private int age;
private String email;
 
// ② No-arg constructor
public Person () {}
 
// ③ All-args constructor
public Person (String name, int age, String email) {
this.name = name;
this.age = age;
this.email = email;
}
 
// ④ Getters — read-only access
public String getName () { return name; }
public int getAge () { return age; }
public String getEmail() { return email; }
 
// ⑤ Setters — controlled write access
public void setName (String name) { this.name = name; }
public void setAge (int age) { this.age = age; }
public void setEmail(String email) { this.email = email; }
 
// ⑥ toString — human-readable representation
@Override
public String toString() {
return "Person{name='" + name + "', age=" + age + "}";
}
}
 
// ⑦ Using the POJO
Person p = new Person("Alice", 30, "alice@mail.com");
p.setAge(31);
System.out.println(p); // Person{name='Alice', age=31}
POJO — Plain Old Java Object
A simple Java class with no dependency on any framework (no Spring, no JPA annotations required).
Widely used to model domain entities like User, Product, Order.
Often called a "model class" or "data class" in projects.
private fields
Fields are hidden from outside code — this is encapsulation.
Outside code must use getters/setters to read or change values.
Protects the object's state from accidental corruption.
No-arg Constructor
A constructor with no parameters. Creates an "empty" object.
Required by many frameworks (e.g. JPA, Jackson JSON) for object creation via reflection.
If you define any constructor, Java no longer auto-generates this one.
All-args Constructor
Allows you to create a fully-populated object in one line.
this.name = name distinguishes the field (left) from the parameter (right).
Libraries like Lombok can auto-generate this with @AllArgsConstructor.
Getter Method
Provides read access to a private field. Naming convention: get + FieldName.
For booleans, use is prefix: isActive() instead of getActive().
Getters let you add logic (e.g. formatting, lazy loading) without breaking callers.
Setter Method
Provides write access to a private field. Naming convention: set + FieldName.
You can add validation inside: if (age < 0) throw new IllegalArgumentException().
Omitting setters makes the object effectively immutable (read-only after construction).
@Override annotation
Tells the compiler this method overrides one from a parent class (here Object.toString()).
Causes a compile error if you accidentally mistype the method name.
Always use @Override — it catches bugs at compile time.
Instantiating a POJO
new Person(...) calls the matching constructor and returns a reference to the new object.
The variable p holds a reference (memory address), not the object itself.
Multiple variables can reference the same object — be careful with mutations.
