Primitive Data Types in Java

Java has 8 primitive types — they hold raw values directly in memory, not references.

public class PrimitiveDemo {
// Integer types
byte age = 25 ; // 8-bit, range: -128 to 127
short year = 2024 ; // 16-bit, range: -32,768 to 32,767
int population = 8000000 ; // 32-bit (most common)
long distance = 9876543210L ; // 64-bit, note the L suffix
 
// Floating-point types
float price = 9.99f ; // 32-bit, note the f suffix
double pi = 3.14159265359 ; // 64-bit, default for decimals
 
// Other types
char grade = 'A' ; // 16-bit Unicode character
boolean isActive = true ; // true or false only
}
byte
Smallest integer type. Uses only 1 byte (8 bits) of memory.
Perfect for storing small numbers or raw binary data.
Range: -128 → 127
short
A 16-bit integer. Rarely used in modern Java — int is almost always preferred.
Range: -32,768 → 32,767
int
The default integer type in Java. 32-bit signed.
Use this unless you have a specific reason to use byte, short, or long.
Range: -2,147,483,648 → 2,147,483,647
long
64-bit integer for very large numbers. Always append L to the literal (e.g. 100L) to avoid compiler errors.
Range: ±9.2 × 10¹⁸
float
32-bit floating-point. Less precise than double. Always use f suffix on literals.
Avoid for financial calculations — use BigDecimal instead.
~7 decimal digits of precision
double
64-bit floating-point. The default for decimal numbers in Java.
Twice the precision of float.
~15 decimal digits of precision
char
Stores a single Unicode character using single quotes.
Internally stored as a 16-bit unsigned integer (0–65,535).
Example: 'A', '€', '\n'
boolean
Can only hold true or false.
Used for all conditional logic and flags. JVM size is implementation-dependent.
Values: true | false
public (access modifier)
The class is visible to all other classes everywhere.
A file can only have one public class, and it must match the filename.
