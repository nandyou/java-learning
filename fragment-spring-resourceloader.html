Spring ResourceLoader

ResourceLoader is Spring's unified API for loading files from any location — classpath, filesystem, URLs, or servlet context — using a simple string prefix.

import org.springframework.core.io.Resource;
import org.springframework.core.io.ResourceLoader;
import org.springframework.stereotype.Service;
import org.springframework.beans.factory.annotation.Autowired;
import java.io.*;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
 
@Service
public class FileReaderService {
 
private final ResourceLoader resourceLoader;
 
@Autowired
public FileReaderService (ResourceLoader resourceLoader) {
this.resourceLoader = resourceLoader;
}
 
// ① Read from classpath (inside the JAR / resources folder)
public String readFromClasspath(String filename) throws IOException {
Resource resource = resourceLoader.getResource(
"classpath:" + filename);
 
try ( InputStream is = resource. getInputStream ()) {
return new String (is.readAllBytes(), StandardCharsets.UTF_8);
}
}
 
// ② Read from external filesystem path
public String readFromFilesystem(String absolutePath) throws IOException {
Resource resource = resourceLoader.getResource(
"file:" + absolutePath);
 
return Files .readString(resource. getFile ().toPath());
}
 
// ③ Read path from application.properties config
@Value ("${app.config.file}")
private Resource configFile; // Spring auto-resolves the prefix
 
public String readConfig() throws IOException {
try ( BufferedReader reader = new BufferedReader (
new InputStreamReader (configFile. getInputStream () ))) {
return reader.lines().collect( Collectors .joining("\n"));
}
}
}

application.properties — configure the resource paths externally:

# Classpath resource (inside JAR)
app.config.file =classpath:config/settings.json
 
# External file on the server disk
app.config.file =file:/etc/myapp/settings.json
 
# Remote URL resource
app.config.file =https://config-server.example.com/settings.json

Load multiple files with wildcard patterns using ResourcePatternResolver:

@Autowired
private ResourcePatternResolver resolver;
 
public void loadAllTemplates() throws IOException {
Resource [] templates = resolver.getResources(
"classpath:templates/*.html" );
 
for (Resource r : templates) {
System.out.println(r. getFilename ());
}
}

Resource (interface)
Spring's abstraction over any file-like thing — classpath entry, filesystem file, URL, or byte array.
Key methods: getInputStream(), getFile(), getURI(), exists(), getFilename()
Your code doesn't need to care where the file comes from — just use the Resource API.
ResourceLoader (interface)
Provides a single method: getResource(String location).
The location string determines the type of Resource returned based on its prefix.
All Spring ApplicationContext implementations also implement ResourceLoader.
classpath: prefix
Loads the file from the application's classpath — typically src/main/resources/.
Works inside packaged JARs — the file is embedded in the archive.
Example: classpath:data/users.csv → src/main/resources/data/users.csv
try-with-resources (InputStream)
The try (InputStream is = ...) syntax automatically closes the stream when done.
Always use this pattern — forgetting to close streams causes resource leaks.
Any class implementing AutoCloseable can be used in try-with-resources.
.getInputStream()
Opens and returns a raw byte stream to the resource's content.
Works for all resource types — classpath, filesystem, and URLs alike.
Use InputStreamReader to read as text, readAllBytes() for binary.
file: prefix
Loads from an absolute path on the server's local filesystem.
Perfect for config files deployed outside the JAR (e.g. /etc/myapp/).
Use relative path without prefix for paths relative to the JVM working directory.
.getFile()
Returns a java.io.File object for filesystem resources.
Throws FileNotFoundException for classpath resources inside JARs — use getInputStream() instead.
Safe rule: always prefer getInputStream() for maximum portability.
@Value injection of Resource
Spring can inject a Resource directly from a property value.
The prefix (classpath:, file:) in the property value tells Spring how to resolve it.
Swapping from classpath to file system requires only a config change — no code change!
Classpath resource in properties
classpath:config/settings.json points to src/main/resources/config/settings.json.
This file is bundled inside the JAR at deployment time.
Good for default configs. Override at runtime with a file: path.
External file in properties
file:/etc/myapp/settings.json reads from the server filesystem.
File must exist at that path at runtime — not bundled in the JAR.
Ideal for production secrets, environment-specific configs, or files that change between deploys.
Remote URL resource
Spring can load a resource over HTTP/HTTPS — useful for config servers or CDN-hosted assets.
Uses standard Java URL connection internally.
For distributed config management, prefer Spring Cloud Config Server.
ResourcePatternResolver
Extends ResourceLoader to support wildcard patterns in paths.
Returns an array of all matching resources.
ApplicationContext implements this — you can also inject it directly.
Wildcard patterns
*.html — all HTML files in the folder.
classpath*: prefix (with asterisk) — scans across all JARs on the classpath, not just the app JAR.
Useful for plugin architectures where multiple JARs contribute config files.
Resource utility methods
.getFilename() — just the file name (settings.json).
.exists() — check before reading to avoid exceptions.
.contentLength() — file size in bytes.
.lastModified() — last-modified timestamp.
Always call .exists() before reading optional resources.
