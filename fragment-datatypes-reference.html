Reference Data Types in Java

Reference types store a memory address pointing to an object — not the value itself.

public class ReferenceDemo {
public static void main(String[] args) {
// String — most used reference type
String name = "Java" ;
String greeting = name. concat ( " is great" );
 
// Arrays — fixed-size collections
int[] scores = { 95 , 87 , 76 };
String[] fruits = new String[ 3 ];
fruits[ 0 ] = "Apple" ;
 
// null — the absence of a reference
String empty = null ;
 
// Wrapper classes — objects wrapping primitives
Integer boxed = 42 ; // autoboxing
int unboxed = boxed; // auto-unboxing
}
}
String
Not a primitive! Strings are objects in Java, instances of java.lang.String.
They are immutable — once created, the value cannot change.
Use StringBuilder for heavy concatenation.
.concat() method
Joins two strings together and returns a new String.
Since Strings are immutable, the original name is unchanged.
Equivalent to: name + " is great"
Array type ( int[] )
Arrays are reference types, even when holding primitives.
Fixed in size — once created, length cannot change.
Access elements with arr[index] (zero-based)
new keyword
Allocates memory on the heap and creates a new object instance.
The variable holds a reference (address) to that memory.
Triggers the object's constructor.
null
Represents the absence of any object reference.
Calling a method on a null reference throws a NullPointerException!
Always check: if (obj != null)
Wrapper Classes
Every primitive has an object equivalent: int → Integer, double → Double, etc.
Java auto-converts between them (autoboxing / unboxing).
Required when using collections like List<Integer>
