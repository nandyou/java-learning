Java List

List is a dynamic, ordered collection from the Java Collections Framework — it grows and shrinks automatically.

import java.util.List;
import java.util.ArrayList;
import java.util.Collections;
 
public class ListDemo {
public static void main(String[] args) {
 
// ① Create an ArrayList
List<String> fruits = new ArrayList<>();
 
// ② Add elements
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Cherry");
fruits.add(1, "Mango"); // insert at index 1
 
// ③ Access and query
String first = fruits. get (0); // "Apple"
int size = fruits. size (); // 4
boolean has = fruits. contains ("Banana"); // true
 
// ④ Update and remove
fruits.set (2, "Grape"); // replace index 2
fruits.remove("Banana"); // by value
fruits.remove(0); // by index
 
// ⑤ Iterate
for (String f : fruits) System.out.println(f);
 
// ⑥ Sort and reverse
Collections.sort (fruits);
Collections.reverse(fruits);
 
// ⑦ Immutable list (Java 9+)
List<Integer> nums = List.of (1, 2, 3);
}
}
List (interface)
List is an interface — it defines behaviour, not implementation.
Common implementations: ArrayList (fast reads), LinkedList (fast inserts).
Program to the interface: List<T> not ArrayList<T>
List<String>
Declaring the variable as the List interface type keeps code flexible.
You can later swap ArrayList for LinkedList without changing every usage.
This is the Dependency Inversion principle in practice.
Generics ( <String> )
The type in angle brackets restricts what the list can hold.
Gives you compile-time type safety — no casting needed on retrieval.
Without generics you'd need: (String) fruits.get(0)
.add(element)
Appends the element to the end of the list.
Use .add(index, element) to insert at a specific position.
ArrayList grows automatically by ~50% when capacity is exceeded.
.get(index)
Returns the element at the given position. Zero-based.
Throws IndexOutOfBoundsException if index ≥ size.
O(1) for ArrayList, O(n) for LinkedList.
.size()
Returns the number of elements currently in the list.
Unlike arrays, this changes as you add/remove items.
Note: it's .size() for collections vs .length for arrays.
.contains(value)
Returns true if the list has at least one element equal to the given value.
Uses .equals() for comparison — not ==.
O(n) linear search — consider a HashSet for faster lookups.
.set(index, value)
Replaces the element at the given index with a new value.
Returns the old element that was there before.
Throws IndexOutOfBoundsException if index is invalid.
.remove()
remove("Banana") — removes by value (first match).
remove(0) — removes by index.
Ambiguity: remove(1) on a List<Integer> removes by index, not value. Use remove(Integer.valueOf(1)) to remove by value.
Collections.sort()
Sorts the list in natural ascending order using Comparable.
For custom order, pass a Comparator: Collections.sort(list, comparator).
Alternatively use: fruits.sort(Comparator.naturalOrder())
List.of() — Immutable List
Creates a list that cannot be modified — no add, remove, or set.
Great for constants and safe API returns.
Available since Java 9. Throws UnsupportedOperationException on modification.
