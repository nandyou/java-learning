if / else Conditions

Conditional statements control which block of code runs based on a boolean expression. Java offers several forms for different situations.

public class ConditionDemo {
public static void main(String[] args) {
 
// ① Simple if
int score = 72;
if (score >= 50) {
System.out.println("Pass");
}
 
// ② if-else
if (score >= 50) {
System.out.println("Pass");
} else {
System.out.println("Fail");
}
 
// ③ if - else if - else (grade classifier)
if (score >= 90) {
System.out.println("A");
} else if (score >= 80) {
System.out.println("B");
} else if (score >= 70) {
System.out.println("C");
} else {
System.out.println("D");
}
 
// ④ Ternary operator — compact if-else
String result = (score >= 50) ? "Pass" : "Fail";
 
// ⑤ Compound conditions
boolean isAdult = true;
boolean hasID = true;
if (isAdult && hasID) {
System.out.println("Entry allowed");
}
 
if (!isAdult || !hasID) {
System.out.println("Entry denied");
}
 
// ⑥ Null check guard
String name = getName();
if (name != null && !name.isEmpty()) {
System.out.println("Hello, " + name);
}
}
}
if ( condition )
The condition must evaluate to a boolean. If true, the block runs; if false, it's skipped entirely.
Java does not accept integers as conditions (unlike C/C++).
if (x) is a compile error in Java. Must be if (x != 0).
else
Runs when the if condition is false. Provides an alternative path.
Only one of the two blocks ever executes — they are mutually exclusive.
Braces {} are optional for single-statement blocks, but always recommended for clarity.
else if
Chains multiple conditions. Each is checked only if all previous conditions were false.
Only the first matching block runs — the rest are skipped.
For many discrete values (days, months), prefer a switch statement instead.
Ternary Operator ( ? : )
Format: condition ? valueIfTrue : valueIfFalse
Returns a value, so it can be used inside expressions and assignments.
Keep it simple — nested ternaries are very hard to read.
&& — Logical AND
Returns true only if both conditions are true.
Short-circuits: if the left side is false, the right side is never evaluated.
Use this to safely chain: obj != null && obj.method()
|| — Logical OR
Returns true if at least one condition is true.
Short-circuits: if the left side is true, the right side is skipped.
! is the logical NOT: !true == false
Null Guard Pattern
Always check for null before calling methods on an object.
Thanks to short-circuit evaluation, if name == null, .isEmpty() is never called (avoids NPE).
Java 8+ alternative: Optional<String> eliminates null checks entirely.
