The do-while Loop

The do-while loop always executes the body at least once — the condition is checked after each iteration, not before.

public class DoWhileDemo {
public static void main(String[] args) {
 
// ① Basic do-while
int count = 1;
do {
System.out.println("Count: " + count);
count++;
} while (count <= 5);
 
// ② Key difference vs while — runs even if false initially
int x = 100;
do {
System.out.println( "Runs once even though 100 > 5" );
} while (x < 5);
 
// ③ Practical: menu-driven input validation
int choice;
do {
System.out.println("1. Start 2. Settings 3. Quit");
choice = scanner.nextInt();
} while (choice < 1 || choice > 3);
 
// ④ Retry logic with do-while
boolean success;
int attempts = 0;
do {
success = tryConnect ();
attempts++;
} while (!success && attempts < 3);
// Tries up to 3 times; stops sooner if success
}
}
do { ... }
The body executes first, unconditionally.
This is the fundamental difference from while: body before condition.
Think: "do this, then check if you should do it again."
} while (condition);
The condition is checked after the body runs.
Notice the semicolon at the end — unique to do-while, easy to forget!
A missing semicolon here is a compile error.
Guaranteed first execution
Even though x = 100 and the condition x < 5 is immediately false, the body still runs once.
This is the defining characteristic of do-while.
A plain while(x < 5) would skip the body entirely.
Menu / Input Validation Pattern
The most classic do-while use case: display a menu, get input, repeat if invalid.
You must show the menu at least once before you can validate the input.
The loop exits only when the user enters a valid choice (1–3).
Retry Pattern
Attempt something, check if it worked, retry if it failed — up to a maximum number of times.
The compound condition !success && attempts < 3 has two exit conditions.
Common in network code, file I/O, and resilience patterns.
